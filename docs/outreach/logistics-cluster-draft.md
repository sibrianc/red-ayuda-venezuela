## Mensaje propuesto

Estimada Andrea, estimada Iris:

Mi nombre es **[nombre completo]** y soy **[rol]** del proyecto independiente Red de
Ayuda Venezuela (`redayudave.org`), una plataforma en desarrollo para organizar y
presentar información humanitaria sobre el terremoto de junio de 2026.

Nuestro objetivo es ayudar a que la población y los equipos autorizados encuentren
información trazable sobre necesidades, recursos, servicios y restricciones de acceso,
sin sustituir los canales oficiales ni publicar datos personales automáticamente.
Cada registro externo permanece privado hasta completar controles de procedencia,
licencia, calidad, privacidad y revisión humana.

Vimos que el Grupo de Trabajo de Logística de Venezuela está activo y que utiliza
herramientas como LogIE y StockHolm/ESUPS para mejorar la visibilidad de capacidades y
existencias. Antes de desarrollar cualquier integración, quisiéramos solicitar su
orientación sobre:

1. fuentes, feeds, APIs o exportaciones que estén autorizados para este propósito;
2. condiciones de uso, atribución, retención, corrección y frecuencia recomendada;
3. posibilidad de recibir únicamente datos agregados sobre acceso, servicios,
   capacidades, suministros y brechas por territorio y período;
4. identificadores, unidades y límites geográficos que debamos conservar para evitar
   duplicados o interpretaciones incorrectas;
5. proceso apropiado para presentar el proyecto y, si corresponde, participar como
   observadores o colaboradores técnicos en la coordinación.

No solicitamos nombres de víctimas, teléfonos, direcciones exactas, expedientes
médicos ni credenciales de sistemas. Tampoco extraeremos datos mediante scraping o
automatización del navegador. Una API autenticada o intercambio entre organizaciones
solo se implementaría mediante un acuerdo explícito, minimización de campos,
idempotencia, auditoría y un entorno privado de staging.

Podemos compartir para su revisión nuestro registro de fuentes, modelo de datos,
política de privacidad y una demostración local antes de recibir información.

Muchas gracias por indicarnos el canal y el proceso adecuados.
